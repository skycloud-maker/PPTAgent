## 1. 이 문서의 목적

PPT Agent가 오류 없이 안정적으로 동작하기 위한 기준과 대응 방법을 정의한다.  
AI 에이전트는 코드 작성 시 이 문서의 기준을 만족해야 한다.

---

## 2. 성능 기준 (SLO)

| 항목 | 목표 |
|------|------|
| 슬라이드 10장 생성 시간 | 60초 이내 |
| API 호출 타임아웃 | 30초 |
| 파일 생성 타임아웃 | 10초 |
| 동시 사용자 | MVP 기준 최대 5명 (Streamlit Cloud 제약) |

---

## 3. 오류 유형별 대응

### 3-1. Claude API 오류

| 오류 유형 | 대응 방법 |
|-----------|-----------|
| 타임아웃 | 최대 2회 자동 재시도 후 사용자에게 오류 표시 |
| 잘못된 JSON 응답 | 재생성 요청 최대 3회, 실패 시 오류 표시 |
| API 키 오류 | 즉시 오류 표시, 관리자 문의 안내 |
| Rate limit 초과 | 10초 대기 후 1회 재시도 |

### 3-2. 파일 생성 오류

| 오류 유형 | 대응 방법 |
|-----------|-----------|
| 렌더링 실패 | 오류 슬라이드 타입 로그 기록 + 사용자에게 재시도 안내 |
| 임시 파일 생성 실패 | 디스크 용량 확인 안내 |
| 다운로드 실패 | 재다운로드 버튼 제공 |

### 3-3. 사용자 입력 오류

| 오류 유형 | 대응 방법 |
|-----------|-----------|
| 필수 항목 미입력 | 인라인 오류 표시, 다음 단계 진행 차단 |
| 지원하지 않는 파일 형식 | 지원 형식 안내 (CSV, JSON) |
| 파일 크기 초과 | 최대 크기 안내 (MVP: 10MB) |

---

## 4. 재시도 정책

```python
# 재시도 기본 원칙
MAX_RETRIES_API = 2       # Claude API 호출 실패 시
MAX_RETRIES_SCHEMA = 3    # JSON Schema 검증 실패 시
RETRY_DELAY_SECONDS = 2   # 재시도 간격
TIMEOUT_API = 30          # API 호출 타임아웃 (초)
```

---

## 5. 사용자 피드백 원칙

- 30초 이상 걸리는 작업은 반드시 진행 상태를 표시한다
- 오류 메시지는 항상 **원인 + 해결 방법**을 함께 표시한다
- 재시도 가능한 오류는 반드시 재시도 버튼을 제공한다
- 시스템 오류(500 등)는 "잠시 후 다시 시도해주세요"로 처리한다
