## 1. 이 문서의 목적

기능 명세서에 담기 어려운 **"왜 이렇게 만드는가"**를 정의한다.  
AI 에이전트가 설계 결정을 내릴 때 이 문서를 판단 기준으로 사용한다.

---

## 2. 제품 철학

### "5분 안에 초안을, 나머지는 사람이"
이 Agent의 목표는 완벽한 장표를 만드는 것이 아니다.  
**사용자가 빈 슬라이드 앞에서 낭비하는 시간을 없애는 것**이다.  
AI는 80점짜리 초안을 빠르게 만들고, 나머지 20점은 사용자가 직접 다듬는다.

### "비개발자도 막힘 없이"
팀 동료 누구나 설명 없이 바로 쓸 수 있어야 한다.  
기능이 많은 것보다 **쓰기 쉬운 것**이 우선이다.

### "믿을 수 있는 도구"
생성할 때마다 결과물이 크게 달라지면 안 된다.  
일관된 디자인, 예측 가능한 구조, 안정적인 동작이 핵심이다.

---

## 3. 기능 추가 판단 기준

새로운 기능 추가 요청이 왔을 때 아래 질문으로 판단한다:

1. **비개발자 팀 동료가 이 기능을 직접 쓸 것인가?**  
   → 아니라면 우선순위 낮춤

2. **이 기능이 없어서 장표를 못 만드는 상황이 생기는가?**  
   → 아니라면 Post-MVP로 분류

3. **이 기능이 기존 흐름을 복잡하게 만드는가?**  
   → 그렇다면 다른 방법을 먼저 찾는다

---

## 4. UX 판단 기준

디자인/UX 결정이 필요할 때 아래 우선순위를 따른다:

1. **명확함** — 사용자가 다음에 무엇을 해야 할지 항상 알 수 있어야 한다
2. **빠름** — 불필요한 클릭과 입력을 줄인다
3. **안전함** — 실수로 데이터가 사라지거나 처음부터 다시 시작하는 상황을 만들지 않는다

## 5. 의도적으로 하지 않는 것

| 하지 않는 것 | 이유 |
|-------------|------|
| 실시간 슬라이드 편집기 제공 | 복잡도 급증, PowerPoint로 충분히 수정 가능 |
| 슬라이드 디자인 완전 자동화 (100점짜리) | 비현실적, 사용자 수정 여지 남기는 것이 목적 |
| 모든 파일 형식 지원 | .pptx 하나에 집중, PDF는 Post-MVP |
| 무한한 템플릿 제공 | 4가지 핵심 UC에 집중, 나머지는 범용 모드로 대응 |
