## 1. 이 문서의 목적

AI가 생성한 코드와 슬라이드의 품질을 판단하는 기준을 정의한다.  
"잘 만들었다"는 주관적 판단이 아니라 **체크리스트로 확인 가능한 기준**을 제공한다.

---

## 2. 코드 품질 기준

### 2-1. 필수 기준 (모두 통과해야 완료)

- [ ] `black` 포매팅 통과
- [ ] `ruff` 린트 오류 없음
- [ ] 모든 공개 함수에 타입 힌트 포함
- [ ] 모든 공개 함수/클래스에 docstring 포함
- [ ] API 키 하드코딩 없음
- [ ] 임시 파일 삭제 처리 포함
- [ ] 모든 외부 API 호출에 try/except 처리

### 2-2. 권장 기준 (가능하면 충족)

- [ ] 함수 길이 50줄 이하
- [ ] 중복 코드 없음 (동일 로직 2회 이상 반복 시 함수로 추출)
- [ ] 매직 넘버 없음 (상수로 분리)

---

## 3. 슬라이드 품질 기준

AI가 생성한 슬라이드 Schema를 검토할 때 사용하는 기준.

### 3-1. 구조 기준

- [ ] 표지 슬라이드가 첫 번째로 위치
- [ ] 슬라이드 수: 최소 5장, 최대 15장
- [ ] 모든 슬라이드에 제목 존재
- [ ] ARCHITECTURE.md에 정의된 슬라이드 타입만 사용

### 3-2. 콘텐츠 기준

- [ ] 불릿 포인트 슬라이드: 항목 5개 이하
- [ ] 표 슬라이드: 열 6개 이하, 행 10개 이하
- [ ] 제목 길이: 20자 이하
- [ ] 본문 한 줄 길이: 50자 이하

### 3-3. 디자인 기준

- [ ] DESIGN.md에 정의된 폰트 사용
- [ ] DESIGN.md에 정의된 색상 팔레트 사용
- [ ] 슬라이드 번호 포함

---

## 4. 완료 기준 체크 방법

각 개발 단위 완료 시 AI 에이전트가 스스로 체크하는 방법:

```
1. 코드 품질: black, ruff 실행 후 결과 확인
2. 슬라이드 품질: SlideSchema 검증 로직 통과 여부 확인
3. 보안: SECURITY.md 체크리스트 항목 확인
4. 완료 기준: PLANS.md 해당 단위의 완료 기준 항목 확인
```
