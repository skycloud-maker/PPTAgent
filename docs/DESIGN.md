## 1. 이 문서의 목적

AI 에이전트가 슬라이드와 UI를 생성할 때 일관된 디자인 기준을 따르도록 정의한다.  
"예쁘게 만들어줘"가 아니라 **구체적인 수치와 규칙**으로 디자인을 통제한다.

## 2. 슬라이드 디자인 시스템

### 2-1. 기본 규격

| 항목 | 값 |
|------|-----|
| 슬라이드 크기 | 와이드스크린 16:9 (33.87cm × 19.05cm) |
| 기본 폰트 | 맑은 고딕 (한국어) / Calibri (영어) |
| 기본 여백 | 상하 1.5cm / 좌우 2.0cm |

### 2-2. 색상 팔레트 (기본 템플릿)

| 역할 | 색상명 | HEX |
|------|--------|-----|
| Primary (강조) | 딥 블루 | `#1E3A5F` |
| Secondary (보조) | 미드 블루 | `#2E75B6` |
| Accent (포인트) | 라이트 블루 | `#BDD7EE` |
| Background | 화이트 | `#FFFFFF` |
| Text (본문) | 다크 그레이 | `#2C2C2C` |
| Text (보조) | 미드 그레이 | `#767676` |
| Border / Divider | 라이트 그레이 | `#D9D9D9` |

> 추후 회사 브랜드 색상으로 교체 가능 (F-08 템플릿 커스터마이징)

### 2-3. 타이포그래피

| 용도 | 폰트 크기 | 굵기 | 색상 |
|------|-----------|------|------|
| 슬라이드 제목 | 28pt | Bold | `#1E3A5F` |
| 섹션 제목 | 22pt | Bold | `#2E75B6` |
| 본문 | 16pt | Regular | `#2C2C2C` |
| 불릿 포인트 | 14pt | Regular | `#2C2C2C` |
| 캡션 / 출처 | 10pt | Regular | `#767676` |
| 슬라이드 번호 | 10pt | Regular | `#767676` |

### 2-4. 슬라이드 타입별 레이아웃 규칙

#### title (표지)
- 배경: Primary 색상 전체 적용
- 제목: 중앙 정렬, 흰색, 36pt Bold
- 부제: 중앙 정렬, 흰색, 18pt Regular
- 날짜/발표자: 우하단, 흰색, 12pt

#### section (섹션 구분)
- 배경: Secondary 색상 좌측 1/3 컬럼
- 섹션 번호 + 제목: 중앙 정렬

#### bullet (불릿)
- 제목 영역: 상단 20% 고정
- 본문 영역: 나머지 80%
- 불릿 최대 5개 (초과 시 슬라이드 분리)
- 들여쓰기 2단계까지만 허용

#### chart (차트)
- 제목 영역: 상단 15%
- 차트 영역: 중앙 75%
- 출처/설명: 하단 10%
- 차트 색상: Primary → Secondary → Accent 순서

#### table (표)
- 헤더 행: Primary 배경 + 흰색 텍스트
- 짝수 행: Accent 배경 (줄무늬)
- 홀수 행: 흰색 배경
- 최대 열 수: 6개 / 최대 행 수: 10개

#### two_column (2단 비교)
- 좌우 동일한 너비로 분할
- 구분선: Border 색상 세로선

## 3. Streamlit UI 디자인 규칙

### 3-1. 컬러 적용
- 주요 버튼: Primary 색상
- 진행 바: Secondary 색상
- 성공 메시지: `#28A745`
- 오류 메시지: `#DC3545`
- 경고 메시지: `#FFC107`

### 3-2. 간격 및 레이아웃
- 섹션 간 간격: `st.divider()` 사용
- 주요 액션 버튼은 항상 우측 정렬
- 입력 폼은 단일 컬럼 (모바일 대응)

### 3-3. 텍스트 원칙
- 모든 UI 텍스트는 한국어
- 버튼 레이블은 동사형으로 ("다음", "생성하기", "다운로드")
- 오류 메시지는 원인 + 해결 방법을 함께 표시
  - 예) "슬라이드 생성에 실패했어요. 잠시 후 다시 시도해주세요."
