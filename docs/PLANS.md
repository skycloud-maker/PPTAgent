## 1. 개발 철학

> **"결정은 사람이, 실행은 AI가"**

- 모든 개발 단위는 사람(Sky)과 AI의 **사전 합의** 후 실행한다
- 합의 내용은 항상 문서에 먼저 기록되고, AI는 문서를 보고 코드를 작성한다
- 한 번의 실행 단위 = **기능 하나가 완성되는 크기**
- 각 단위 완료 후 Sky가 직접 확인 & 승인 → 다음 단위로 이동

---

## 2. 개발 사이클

```
[1] 회의 & 합의
    Sky + AI가 다음 기능에 대해 논의
    → 무엇을 만들지, 어떻게 만들지 결정
         ↓
[2] 문서 업데이트
    합의 내용을 관련 문서에 기록
    (ARCHITECTURE / PRODUCT_SPECS / FRONTEND 등)
         ↓
[3] AI 코드 실행
    AI가 문서를 기반으로 코드 작성 & 실행
         ↓
[4] 검토 & 승인
    Sky가 결과 확인
    → 승인 시 다음 단위로 / 수정 필요 시 [3]으로 재실행
```

## 4. 첫 번째 실행 단위 상세 (1-A)

> 다음 회의에서 합의 후 AI가 실행할 내용 예시

### 1-A: 프로젝트 구조 세팅

**합의 체크리스트** (실행 전 Sky와 확인):
- [ ] 프로젝트 루트 디렉토리 이름
- [ ] Python 버전
- [ ] 패키지 관리 도구 (uv / pip / poetry)
- [ ] 환경변수 관리 방식 (.env / secrets)
- [ ] Streamlit Cloud 배포 여부 (초기부터 or 나중에)

**AI 실행 범위**:
```
ppt-agent/
├── app.py                  # Streamlit 진입점
├── core/
│   ├── llm/
│   │   ├── interface.py    # LLMInterface 추상 클래스
│   │   └── claude.py       # ClaudeAdapter
│   ├── schema.py           # SlideSchema 정의 (Pydantic)
│   └── renderer.py         # python-pptx 렌더러
├── templates/              # 템플릿 정의 파일
├── .env.example
├── requirements.txt
└── README.md
```

**완료 기준**:
- `app.py` 실행 시 Streamlit 앱이 브라우저에서 열림
- 폴더 구조가 위와 일치함
- `.env.example`에 필요한 환경변수 목록이 정리됨

## 5. 실행 원칙

1. **문서 먼저**: 코드 작성 전 항상 관련 문서가 최신 상태여야 한다
2. **한 단위씩**: 이전 단위 승인 전 다음 단위 실행 금지
3. **합의 체크리스트**: 각 단위 실행 전 Sky와 확인할 항목을 명시
4. **완료 기준 명확화**: "완료"의 정의를 사전에 합의하여 모호함 제거
5. **롤백 기준**: 완료 기준 미달 시 재실행, 2회 이상 실패 시 설계 재검토
